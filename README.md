🧠 Как работает
1. cv2.selectROI() → ROI столика
2. YOLOv8s (conf=0.1) → люди в ROI
3. Гистерезис 1.2с → устойчивое состояние
4. FRAME_SKIP=3 → ×3 быстрее!
5. События: approach/leave → статистика

Минусы реализации
1. в тз было написано использовать selectROI для выделения области столика но из-за неудобных ракурсов и обрезки видео появляются лишние места или загорождения из-за которых появляются фантомные занятия столиков, можно использовать четырехугольник вместо прямоугольника → cv2.polylines(roi_corners) + cv2.fillPoly(mask)
2. Медленная обработка длинных видео
Изза того что обрабатовалось локально на моем ноутбуке, обработка составляла около 20 минут(где-то 2-3 кадра в секунду), поэтому был добавлен FRAME_SKIP=3 для ускорения процесса (так-же можно перебросить модель на гпу)

Визуализация:
🟢 Зеленый = пустой стол
🔴 Красный = занят

🐛 Проблемные случаи
1. Изза того что прямоугольник выходит изза стола то рядом стоящие люди попадают в столик и он становится занятым хотя в нем никто не сидит
